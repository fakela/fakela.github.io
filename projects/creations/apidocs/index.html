API documentation

Context

Write API documentation for API users, such as API reference docs to enable Mindee users understand how the API works.

Problem

The API documentation needed to have contain code samples, use cases, authentications, and other resources that will enable a developer to right off the bat start using the Mindee API.

As a result, they needed a Technical Writer who had knowledge in writing good API documentation, understanding data formats and API design, have a knowledge of REST API, and how to generally write for a developer audience.

Contribution

Over the course of the two projects, I worked on:

  • Providing use case examples that explain the most effective ways to use the Mindee API.
  • Provide thorough descriptions of the Mindee API’s functions and how it interacts with the application.
  • Review documents written by developers / PMs for a particular API and suggest changes / seek clarity to externalize the same documents.
  • Organize documentation so it is easy to use and search
